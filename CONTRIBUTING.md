# Contributing to TwinDB

We welcome contributions to TwinDB! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/twindb.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Run tests: `python -m pytest`
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- Follow PEP 8 for Python code
- Use type hints for function signatures
- Write docstrings for public functions and classes
- Keep functions focused and small

## Testing

- Add tests for new features
- Ensure all tests pass before submitting a PR
- Run benchmarks if performance-critical code is changed

## Reporting Issues

When reporting issues, please include:
- Python version
- Operating system
- Steps to reproduce
- Expected vs actual behavior
- Error messages (if any)

## Questions

For questions, please open a GitHub issue with the "question" label.

Thank you for contributing!
